# C-008 / C-009 / C-010 — Interpolation repr (the Almide-literal form)

> Prose companion to contracts C-008, C-009, C-010 in
> [contracts.toml](contracts.toml). Ported from the `compound_repr_*` /
> `recursive_generic_repr_interp` fixture headers and PR #379 — not re-derived.

`"${value}"` renders a value back to a string. For the shapes below it renders
the **Almide-literal form** — the same source you would write to reconstruct the
value, the same format `deriving Repr` produces — recursively composed and
**byte-identical on native and wasm**. Native uses the `AlmideRepr` trait; wasm
uses the recursive `rt` repr emitters / per-type `__repr_*` functions. Which
types take the repr path (vs the plain Display path) is gated by `ty_needs_repr`.

## Per-type repr format

| Kind | Almide-literal repr | Contract |
|------|---------------------|----------|
| List | `[a, b, c]`, empty `[]` | C-008 |
| Map | `["a": 1, "b": 2]` (brackets, insertion order — see C-013) | C-008 |
| Set | `set.from_list([3, 1, 2])` (dedup, insertion order — see C-014) | C-008 |
| Tuple | `(1, "two", true)` (heterogeneous) | C-008 |
| Option | `some(42)` / `none` | C-008 |
| Result | `ok(7)` / `err("boom")` | C-008 |
| String (inside a container) | quoted with `"`, `\n`, `\t`, `\\` escaped | C-008 |
| Float (inside a container) | shortest round-trip (C-023), e.g. `0.30000000000000004` | C-008 |
| Record | `Point { x: 1, y: 2 }` (declared field order) | C-009 |
| Tuple-variant | `Click(10, 20)` | C-009 |
| Record-variant | `Scroll { dy: 3 }` | C-009 |
| Nullary variant | `Empty` | C-009 |
| Anonymous record | `{ x: 1, y: 2 }` — **no type name, fields SORTED by name** | C-009 |
| Self-recursive ADT | `Node(Leaf(1), Leaf(2))` via `__repr_Tree` (recurse by CALL) | C-010 |
| Self / mutually-recursive record | `Node { val: 1, kids: [...] }` | C-010 |
| Generic record / variant | keyed by INSTANTIATION: `__repr_Tree_Int` vs `__repr_Tree_String` | C-010 |

## The three subtle invariants the fixtures lock

1. **Anonymous records sort fields by name on BOTH targets** (C-009). An
   anonymous record has no declared type name, so there is no canonical field
   order to inherit. Native synthesizes an `AlmdRec_*` struct that stores fields
   sorted by name; the wasm walk sorts to match. `{ y: 2, x: 1 }` and
   `{ x: 1, y: 2 }` both repr as `{ x: 1, y: 2 }`.

2. **Recursion is by CALL on the finite value, not by compile-time type
   expansion** (C-010). A self-recursive `Tree = Leaf(Int) | Node(Tree, Tree)`
   cannot be repr'd by an inline type-driven walk on wasm — the type graph is
   infinite. It routes through `__repr_Tree`, which recurses into itself on the
   *value*, terminating on the finite data.

3. **Generic repr fns are keyed by INSTANTIATION, not by bare name** (C-010).
   The repr fn for `Tree[T]` used to be monomorphic by bare name (`__repr_Tree`),
   so it read the `T` payload as a raw `TypeVar` and rendered `Leaf(T {  })`.
   The fix keys it by the concrete instantiation (`__repr_Tree_Int`,
   `__repr_Tree_String`, `__repr_Tree_List_Int` …), substituting the site's type
   args through the recursive references (`Node`'s children are `Tree[T]` -> the
   same instantiation). `Tree[Int]` AND `Tree[String]` in one program is the
   mono-by-name collision case that previously corrupted one of the two.

## Display vs repr — the `.0` distinction (C-011)

A **bare** integer-valued float in interpolation (`"${f}"`) renders via the
**Display** form, which DROPS the `.0` suffix. But a float **inside a container**
takes the repr path and `float.to_string` keeps the `.0`. Both forms are
byte-identical across targets; the distinction is which path the value is on, and
it is locked by C-011 alongside these contracts.
