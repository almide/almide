# E007 — `fan` block outside effect fn

`fan { ... }` (concurrent execution block) can only be used inside an `effect fn`.

## Common cases

```almide
fn parallel_work() -> Int = fan { a; b; c }    // E007: fan in pure fn
```

## Fix

- Convert the enclosing function to `effect fn`:
  ```almide
  effect fn parallel_work() -> Int = fan { a; b; c }
  ```

## Why

`fan` runs branches concurrently and aggregates results. Concurrency is an effect (timing, order), so it requires effect context.
