# E007 — `fan` block outside effect fn

`fan { ... }` (concurrent execution block) can only be used inside
an `effect fn`. Concurrency is an effect.

## Common cases

```almide
fn parallel_work() -> Int = fan { a; b; c }    // E007: fan in pure fn
```

## Diagnostic

```
error[E007]: fan block can only be used inside an effect fn
  hint: Mark the enclosing function as `effect fn`
```

## Fix

- Convert the enclosing function to `effect fn`:
  ```almide
  effect fn parallel_work() -> Int = fan { a; b; c }
  ```

## Why

`fan` runs branches concurrently and aggregates their results. The
timing, scheduling, and any I/O inside branches are observable effects.
Forcing `fan` to live in `effect fn` keeps the effect boundary honest —
a pure `fn` signature must mean "no observable ordering or I/O
dependency."
