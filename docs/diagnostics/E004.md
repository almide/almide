# E004 — Wrong number of arguments

A function call passes too few or too many arguments.

## Common cases

```almide
list.map(xs)                  // missing closure
math.pow(2)                   // pow expects (base, exp)
fn f(a, b) = a + b ; f(1)     // missing b
```

## Fix

- Check the function signature in `docs/CHEATSHEET.md` or `docs/STDLIB-SPEC.md`.
- For optional params: stdlib uses positional only — pass `none` explicitly if you need to skip.
- For methods via UFCS: remember the receiver (first arg) is the value before the dot.
