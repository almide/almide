# E004 — Wrong number of arguments

A function call passes too few or too many arguments.

## Common cases

```almide
list.map(xs)                  // missing closure
string.join(xs)               // missing separator
math.pow(2)                   // pow expects (base, exp)
fn f(a, b) = a + b ; f(1)     // missing b
```

## `try:` snippet with signature placeholder

```
error[E004]: string.join() expects 2 argument(s) but got 1
  hint: Check the number of arguments
  try:
      // string.join() takes 2 arg(s) — you passed 1
      string.join(<list: List[String]>, <sep: String>)
```

The snippet lists every parameter with its name and type as
`<name: Type>` placeholders. LLMs fill in the placeholders in the next
retry, converging to a valid call.

## Fix

- Check the function signature. Fast lookup:
  - `almide ide doc <module>.<fn>` — signature only
  - `almide ide outline @stdlib/<module>` — full module listing
  - `docs/CHEATSHEET.md` — common cases
- For optional params in stdlib: positional only — pass `none`
  explicitly if you need to skip an early optional to reach a later
  required one.
- For UFCS / method-style chaining (`x |> f`): remember the receiver
  (first arg) is the value before the pipe.
