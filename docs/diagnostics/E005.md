# E005 — Argument type mismatch (constructor / function call)

A variant constructor, record builder, or function call received an
argument whose type doesn't match the declared parameter.

## Common cases

```almide
type Point = Pt(Int, Int)
let p = Pt("x", "y")                  // expected Int, got String

type Box = { value: Int }
let b = Box { value: "abc" }          // record field type mismatch

fn gt2(n: Int) -> Bool = n > 2
let r = gt2("5")                      // expected Int, got String
```

## Fix — conversion helpers

When the expected type differs only by primitive conversion, insert
the stdlib conversion:

| Got | Expected | Conversion |
|---|---|---|
| `Int` | `String` | `int.to_string(n)` |
| `String` | `Int` | `int.parse(s)?` (effect fn) or `int.parse(s).unwrap_or(0)` |
| `Int` | `Float` | `int.to_float(n)` |
| `Float` | `Int` | `float.to_int(n)` (truncating) or `math.round(n)` then to_int |
| `List[A]` | `Option[A]` (single element) | `list.first(xs)` |
| `Option[A]` | `A` | `x ?? default` or `x!` (effect fn only) |
| `Result[A, E]` | `Option[A]` | `r?` (inside a fn returning Option) |
| `Result[A, E]` | `A` | `r!` (effect fn only; propagates Err) |

The diagnostic's `hint:` line often names the specific conversion when
the mismatch is a known primitive pair.

## Fix — structural

- For variants with no payload, call as `MyVariant` (no parens).
  Calling `MyVariant()` is a function call of zero args and will not
  match a zero-arg variant.
- For records, all required fields must be present. Use
  `Record { ..old, changed_field: new_value }` spread syntax to copy
  an existing record with a field override.
- For generic functions, sometimes `E005` hides a checker limitation
  (inference did not propagate a type variable). Add an explicit type
  arg: `fn_call[TypeArg](args)`.
