# E017 — Record construction on an enum type name

A record literal `{ field: value, ... }` was prefixed with the name of
an **enum type** (a `type T = A | B { ... }` declaration) rather than
the name of a record type or a specific record-bearing case. An enum
type has no fields of its own — only its individual cases do — so the
construction is meaningless.

This is a hard error on **both** targets. It used to diverge: the Rust
target leaked `rustc` `error[E0574]: expected struct, variant or union
type, found enum` (wrapped as an "Almide bug"), while the WASM target
compiled and silently mis-constructed the value (reading the named
field as garbage). Rejecting it at type-check keeps the two targets in
agreement and produces an actionable message.

## Common case

```almide
type V = Tag(Float) | Named { who: String }

let v = V { who: "x" }
// error[E017]: cannot construct enum type 'V' with record syntax
```

## Fix

Construct the specific record-bearing case instead of the enum type:

```almide
let v = Named { who: "x" }   // the case, not the enum
```

If the enum has no record-bearing cases, pick the case you meant and
supply its payload directly (`Tag(1.0)`).
