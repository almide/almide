# E002 — Undefined function

Function called by name is not in scope.

## Common cases

```almide
foo()                                  // foo is not defined
list.lengt(xs)                         // typo: should be list.len
my_module.something()                  // forgot to import my_module
```

## Cross-language idiom hallucinations (with `try:` snippets)

### Method-call syntax on a stdlib type (`x.to_upper()`)

```almide
let s = "hi"
s.to_upper()                           // ← method-call is not supported
```

Diagnostic:

```
hint: Almide doesn't use method-call syntax. Write `string.to_upper(x)` (or `x |> string.to_upper`).
try:
    // x.to_upper()  →  string.to_upper(x)
    string.to_upper(x)
```

### `int.gt(a, b)` / `int.lt` / `int.eq` / similar comparison functions

Almide uses operators for comparison; there is no `int.gt`.

```
try:
    // Almide uses operators, not comparison functions:
    //   int.gt(a, b)   →  a > b
    //   int.lt(a, b)   →  a < b
    //   int.gte(a, b)  →  a >= b
    //   int.lte(a, b)  →  a <= b
    //   int.eq(a, b)   →  a == b
    //   int.neq(a, b)  →  a != b
```

`almide fix` auto-applies this rewrite (`int.gt(n, 0)` → `n > 0`).

### `int.sqrt(n)` (doesn't exist — only `float.sqrt`)

```
try:
    // Almide has float.sqrt; int.sqrt doesn't exist.
    // Convert → sqrt → (optionally) convert back:
    let root_f = float.sqrt(int.to_float(n))       // Float
    let root_i = float.to_int(root_f)              // Int (truncates)
```

### Other common renames

| User wrote | Almide is | Auto-fixable? |
|---|---|---|
| `list.size` / `list.length` / `list.count` | `list.len` | via fuzzy |
| `list.push(xs, x)` | `xs + [x]` | hint only (expression) |
| `list.head(xs)` / `list.tail(xs)` | `list.first(xs)` / `list.drop(xs, 1)` | via fuzzy (head) |
| `string.length` / `string.substring` | `string.len` / `string.slice` | via fuzzy |
| `string.to_uppercase` | `string.to_upper` | via fuzzy |
| `string.from_chars` | `list.join(chars, "")` | hint only |

## Fix (general)

- Check spelling. The diagnostic includes a "Did you mean ...?"
  suggestion when fuzzy match lands; `almide fix` auto-applies the
  clean renames (module-qualified `X` → `Y`).
- For stdlib: Tier 1 modules (`string`, `list`, `int`, `float`, `map`,
  `set`, `value`, `env`, `option`, `result`) are auto-imported.
  Tier 2 (`json`, `fs`, `http`, `regex`, `datetime`, `bytes`, `matrix`,
  `process`, `testing`, `random`, `io`) need explicit `import`.
- For your own modules: `import self.<sub>` and call as `<sub>.fn()`,
  or selective import `import self.<sub>.{fn1, fn2}`.
- Run `almide ide outline @stdlib/<module>` to see the authoritative
  API list for a stdlib module.
