# E028 — main() takes no parameters

`main` is the program entry point and takes no parameters. The parameter form
(`fn main(args: List[String])`) typechecks structurally but no codegen leg
wires the argument: the native driver calls the entry 0-ary, and the wasm
`_start` glue likewise — both used to fail downstream with a compiler-blaming
error ("codegen produced invalid Rust — this is an Almide bug") for a
signature the language never supported. The checker now rejects it up front.

```almide
effect fn main(args: List[String]) -> Result[Unit, String] = ok(())   // ← E028
```

## Fix

Read program arguments with `env.args()` inside the body (`env` is a Tier 2
stdlib module — add the import):

```almide
import env

effect fn main() -> Result[Unit, String] = {
  let args = env.args()
  println(int.to_string(list.len(args)))
  ok(())
}
```
