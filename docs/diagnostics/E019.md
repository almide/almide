# E019 — Ambiguous constructor

A bare variant constructor name is declared in **more than one** variant
type, so a use of it by itself is ambiguous — the checker cannot tell
which type you meant. This most often happens when a local type and a
dependency package both declare a constructor with the same name.

Almide resolves the name to the first declared candidate so checking can
continue, but reports the conflict instead of silently constructing the
wrong type — which would otherwise surface much later as a cryptic
generated-Rust error (`E0769`/`E0308`).

## Common cases

```almide
type Cmd   = | Stop | Send(Int)
type Event = | Tick | Send(Int)   // `Send` now lives in two types

fn main() -> Unit = {
  let x = Send(5)   // E019: ambiguous constructor 'Send': declared in Cmd and Event
  println("${x}")
}
```

## Fix

Rename the constructor in one of the types so the name is unique. A
qualified `Type.Ctor` form is not yet supported, so renaming is the way
to disambiguate.

```almide
type Cmd   = | Stop | Send(Int)
type Event = | Tick | Emit(Int)   // renamed `Send` → `Emit`

fn main() -> Unit = {
  let x = Send(5)   // unambiguous: only Cmd declares `Send`
  println("${x}")
}
```
