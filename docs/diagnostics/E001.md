# E001 — Type mismatch

The type of an expression doesn't match what the surrounding context expects.

## Common cases

```almide
let x: Int = "hello"             // expected Int, got String
fn f() -> Int = "abc"            // body type doesn't match return type
list.map(xs, (x) => x + "!")     // closure return type conflicts with list element
```

## Fix

- Adjust the literal/value to match the expected type.
- Change the explicit type annotation to match the actual expression.
- Convert with stdlib helpers: `int.to_string`, `int.parse(s)?`, `float.from_int`, etc.
