# E027 — fan.timeout was removed

`fan.timeout(ms, thunk)` was removed in 0.29.0 (contract C-006). A wall-clock
timeout has no portable cross-target meaning: native could enforce it with a
real thread and `recv_timeout`, but wasm has no clock, scheduler, or threads,
so the thunk always ran to completion and the timeout never elapsed. It was
also the sole stdlib surface whose result was not a function of the program
and its inputs — whether the deadline fired depended on machine load, even
between two native runs. Both properties contradict the language's core
promise: byte-identical observable output across targets.

```almide
effect fn main() -> Unit = {
  let r: Result[Int, String] = fan.timeout(5000, () => slow())   // ← E027
}
```

## Fix

Enforce deadlines at the host boundary that invokes the program:

```bash
timeout 5 ./app          # POSIX
wasmtime run app.wasm    # + host-side deadline of your choice
```

Inside Almide, run the work directly — every fan combinator is deterministic
by list order and stays inside the equivalence guarantee:

```almide
effect fn main() -> Unit = {
  let r: Result[Int, String] = fan.race([() => slow()])
}
```
