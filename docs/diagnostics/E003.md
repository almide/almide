# E003 — Undefined variable

A name used as a value is not bound in the current scope.

## Common cases

```almide
fn f() -> Int = x                      // x is not declared
match v { ok(value) => valeu }         // typo: 'valeu' instead of 'value'
let x = y                              // y is not bound at this point
json.stringify(v)                      // forgot `import json`
```

## Specialized sub-patterns (with `try:` snippets)

### Forgot `import json` / `import fs` / etc.

For Tier 2 stdlib modules that require explicit import, the first use
of `<module>.<fn>` emits:

```
error[E003]: undefined variable 'json'
  hint: Add `import json` (stdlib: JSON parsing and querying)
  try:
      import json
```

`almide fix` auto-applies this (calls `auto_imports` from the
formatter). No user action needed — the harness just runs `almide fix`
before retry.

### Typo of an in-scope name

When a fuzzy match lands:

```
error[E003]: undefined variable 'lenth'
  hint: Did you mean `length`?
  try:
      // lenth  →  length
      length
```

## Fix (general)

- Declare with `let` or `var` before use.
- Check spelling against pattern-bound names (`match v { ok(value) => ... }`
  binds `value`, not `val` or `valeu`).
- For let bindings inside expressions, ensure they're in a
  `Block { ... }` that contains the use. A bare `let x = 1` at top
  level is a `TopLet` decl; inside a fn body you need `{ let x = 1; x }`
  or equivalent.
- For Tier 2 stdlib: add `import <module>` at the top (or run
  `almide fix` to do it automatically).
