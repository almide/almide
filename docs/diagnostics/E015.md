# E015 — Possible stdlib reimplementation

A top-level `fn` has the **same signature** as a stdlib function
whose name is within edit distance ≤ 2 of yours. Almide flags this
as a warning (not an error) so compilation still succeeds — the
lint is a nudge toward the idiomatic one-liner, not a gate.

The `try:` snippet in the diagnostic is a **delegation shim** (`fn
your_name(args) -> T = stdlib_mod.stdlib_fn(args)`), not a rewrite
of your body. Decide whether the existing body has a reason to
differ from the stdlib version before accepting the suggestion.

## Common case

```almide
// ~30 lines of algorithm:
fn binary_search(xs: List[Int], target: Int) -> Option[Int] = {
    let lo = 0
    let hi = list.len(xs) - 1
    // ... loop implementation ...
}
// warning[E015]: has the same signature as stdlib `list.binary_search`
```

## Fix

Either accept the delegation shim:

```almide
fn binary_search(xs: List[Int], target: Int) -> Option[Int] =
    list.binary_search(xs, target)
```

Or rename your fn if the algorithm is intentionally different:

```almide
fn my_search_descending(xs: List[Int], target: Int) -> Option[Int] = ...
```

## When E015 does NOT fire

The lint is strict on all three axes — a miss on any suppresses the
suggestion, so false positives are near-zero:

- **Name distance > 2** — `fn find_by_binary_search` doesn't match.
- **Different param types** — `fn binary_search(xs: List[Float], ...)`
  doesn't match the `List[Int]` stdlib signature.
- **Different return shape** — `fn binary_search(...) -> Int` returning
  `-1` on miss doesn't match `-> Option[Int]`.
- **Param arity** — extra or missing params exclude the candidate.

The stdlib side may be generic (`binary_search[T: Ord]`) and the
user side monomorphic (`List[Int]`) — that still counts as a match;
asymmetric `TypeVar`-wildcard is deliberate.

## Why warning, not error?

There's no correctness issue — your fn works. The measurement
hypothesis (`docs/roadmap/done/reimpl-lint.md`) is that LLM retry
loops benefit from seeing a concrete alternative at diag time.
Flipping to error would break legitimate specializations.
