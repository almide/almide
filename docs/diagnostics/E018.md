# E018 — Empty collection with an uninferable element type

An empty collection literal (`[]`, `[:]`), or a generic collection
constructor that takes no element-bearing argument (`set.new()`,
`map.new()`, `list.with_capacity(n)`), was used in a position where
**no surrounding context pins its element type**. An empty collection
carries no element to infer from, so the element type is undecidable.

Almide follows **Rust and Swift**: an undecidable empty collection is a
compile error demanding an explicit type annotation — it is never
silently defaulted to some arbitrary element type. Observability is
irrelevant, exactly as in Rust/Swift: `for _ in []` is an error even
though the element is never read.

This is a hard error on **both** targets because it is raised in the
frontend type checker, before code generation. It used to diverge:
the Rust target leaked `rustc` `error[E0282]: type annotations needed`
on the emitted `Vec::<_>::new()` (wrapped as an "Almide bug"), while
the WASM target — which carries no element type — compiled and ran.
Rejecting it at type-check keeps the two targets in agreement and
produces an actionable message.

## Common cases

```almide
let n: Int = list.len([])            // element of `[]` never constrained
let c: Int = set.len(set.new())      // element of the set never constrained
for _ in [] { }                      // iterated element type undecidable
```

```text
error[E018]: cannot infer the element type of empty list `[]`
```

## Fix

Annotate the element type. The most reliable form is a typed binding:

```almide
let xs: List[Int] = []
let n: Int = list.len(xs)

let s: Set[Int] = set.new()
let c: Int = set.len(s)

let empty: List[Int] = []
for _ in empty { }                   // for over an explicitly-typed list
```

For a bare list literal in call-argument position you can also annotate
it inline:

```almide
let n: Int = list.len([]: List[Int])
```

If the empty collection is a field value, a default field value, or an
assignment, the element type is taken from the declared field / variable
type — no extra annotation is needed there:

```almide
type Bag = { items: List[Int] }
let b = Bag { items: [] }            // element inferred as Int from the field
```
