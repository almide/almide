# E029 — unknown type name

Every type name mentioned in an annotation or a record literal must be a
declared (or imported) type. An undeclared name previously flowed through
unification unconstrained — `let xs: List[Inner] = []` or `Inner { x: 1 }`
with no `type Inner` typechecked, then failed the native build with rustc
`E0412`/`E0422`/`E0425` (check accepted, build failed — the acceptance-parity
gap differential-fuzz seed 20260718 index 940 hit with a mutated-away `type`
declaration).

```almide
fn main() -> Unit = {
  let xs: List[Inner] = []          // ← E029: unknown type 'Inner'
  let n = Inner { x: 1, y: 10 }     // ← E029: unknown type 'Inner'
}
```

## Fix

Declare the type (or fix the spelling / add the missing import):

```almide
type Inner = { x: Int, y: Int }

fn main() -> Unit = {
  let xs: List[Inner] = [Inner { x: 1, y: 10 }]
  println("${list.len(xs)}")
}
```

Generic parameters are immune by construction: an in-scope generic (`fn
f[T](x: T)`) resolves to a type variable at annotation time, never a nominal
name. The built-in dynamic `Value` type is likewise always in scope.
