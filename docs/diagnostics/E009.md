# E009 — Reassignment to immutable binding

`let x = ...` creates an immutable binding. Use `var` for mutable bindings.

## Common cases

```almide
let count = 0
count = count + 1            // E009: 'count' is immutable
```

## Fix

- Use `var` for mutability:
  ```almide
  var count = 0
  count = count + 1          // OK
  ```
- Prefer functional style: produce a new value instead of mutating.
  ```almide
  let count = 0
  let count2 = count + 1
  ```

## Why

Immutability is the default to maximize LLM modification survival. Explicit `var` signals intent to mutate.
