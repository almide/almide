# E006 — Effect isolation: pure fn calls effect fn

A non-effect function (`fn`) tried to call an effect function (`effect fn`). This breaks Almide's effect tracking.

## Common cases

```almide
effect fn read_text(path: String) -> String = ...

fn pure_fn() -> String = read_text("a.txt")    // E006: pure calls effect
```

## Fix

- Mark the calling function as `effect fn`:
  ```almide
  effect fn caller() -> String = read_text("a.txt")
  ```
- Or pull the effect call up to a context that is `effect fn`, and pass the result down.

## Why

`effect fn` types track side effects and propagate `Result<T, String>` returns automatically. Letting a `fn` call an `effect fn` would silently lose this information.
