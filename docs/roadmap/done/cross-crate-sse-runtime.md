# Cross-crate SSE runtime functions

> **Priority: high** — blocks almai from being used as a dependency.
> **Status: root cause identified, fix is 1 line**

## Problem

When a crate depends on almai, the SSE streaming runtime functions are referenced but not emitted:

```
error[E0425]: cannot find function `almide_rt_sse_anthropic_messages` in this scope
error[E0425]: cannot find function `almide_rt_sse_openai_chat` in this scope
```

## Root Cause

`http.anthropic_streaming_call` and `http.openai_streaming_call` are stdlib functions in the `http` module, but their codegen emits calls to `almide_rt_sse_*` functions which live in the `sse` runtime module.

`RUNTIME_DEPS` in `crates/almide-codegen/src/generated/rust_runtime.rs` is missing the `http → sse` dependency:

```rust
pub const RUNTIME_DEPS: &[(&str, &[&str])] = &[
    ("json", &["value"]),
    ("sse", &["json"]),
    // MISSING: ("http", &["sse"]),
];
```

When `http` is in `needed` (because almai uses it), `sse` is NOT transitively pulled in.

## Fix

Add `("http", &["sse"])` to `RUNTIME_DEPS`:

```rust
pub const RUNTIME_DEPS: &[(&str, &[&str])] = &[
    ("json", &["value"]),
    ("http", &["sse"]),
    ("sse", &["json"]),
];
```

This is auto-generated by `build.rs` — if `build.rs` scans runtime source for cross-module references, make sure `runtime/rs/src/http.rs` references to `almide_rt_sse_*` are detected.

## Exit criteria

```almide
import almai
test "simple" {
  let msg = almai.user("hello")
  assert_eq(msg.role, "user")
}
```

compiles without SSE errors.
