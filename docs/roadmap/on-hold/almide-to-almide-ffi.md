# Almide-to-Almide FFI via almide-lander

## Thesis

almide-lander は Almide モジュールを 20 言語に export できる。その 21 番目のターゲットとして **Almide 自身** を追加する。コンパイル済み Almide ライブラリを、別の Almide プロジェクトから shared library 経由で呼べるようにする。

```
almide-lander (today):
  mylib.almd → .so/.dylib → Python/Go/Swift/... から呼べる

almide-lander + Almide target:
  mylib.almd → .so/.dylib → Almide から呼べる（@extern wrapper 生成）
```

## Why

1. **Pre-compiled パッケージ配布** — ソースを配布せずバイナリだけ共有できる。企業内の proprietary ライブラリに最適。
2. **Capability isolation** — 呼ばれる側は独立してコンパイルされた binary。capability scope が分離される。
3. **almide-lander の既存インフラ活用** — interface.json 抽出、shared library ビルド、バインディング生成の仕組みが全て再利用できる。新しいのは Almide 向け codegen だけ。
4. **WASM Component Model への踏み石** — 今は shared library FFI、将来は WASM module 間呼び出しに進化。同じ interface.json が基盤。

## Architecture

```
producer (mylib)                    consumer (app)
─────────────────                   ──────────────
mylib.almd                          app.almd
  │                                   │
  ├─ almide compile --json            ├─ import mylib
  │  → interface.json                 │    (生成された @extern wrappers)
  │                                   │
  ├─ almide --target rust --repr-c    ├─ mylib.distance(a, b)
  │  → source.rs                      │    → @extern call → .so/.dylib
  │  → cargo build                    │
  │  → libalmide_mylib.dylib          │
  │                                   │
  └─ almide-lander --lang almide      │
     → almide_mylib/                  │
        ├── types.almd    (型定義)     │
        └── mod.almd      (@extern)   │
```

## Generated Code Example

**元の mylib.almd:**
```almide
type Point = { x: Float, y: Float }
type Shape = Circle(Float) | Rect(Float, Float)

fn distance(a: Point, b: Point) -> Float = {
  let dx = a.x - b.x
  let dy = a.y - b.y
  math.sqrt(dx * dx + dy * dy)
}

fn area(shape: Shape) -> Float = match shape {
  Circle(r) => math.pi() * r * r,
  Rect(w, h) => w * h,
}
```

**生成される almide_mylib/mod.almd:**
```almide
// Auto-generated by almide-lander --lang almide
// Do not edit manually.

type Point = { x: Float, y: Float }
type Shape = Circle(Float) | Rect(Float, Float)

@extern(lib: "almide_mylib", "almide_mylib_distance")
fn distance(a: Point, b: Point) -> Float

@extern(lib: "almide_mylib", "almide_mylib_area")
fn area(shape: Shape) -> Float
```

**consumer の app.almd:**
```almide
import mylib

fn main() -> Unit = {
  let a = mylib.Point { x: 0.0, y: 0.0 }
  let b = mylib.Point { x: 3.0, y: 4.0 }
  println(float.to_string(mylib.distance(a, b)))  // 5.0
}
```

## What almide-bindgen Needs

almide-bindgen に `almide` ターゲットを追加:

1. **型マッピング** — identity mapping（Almide → Almide なので型が同じ）
2. **関数 wrapper 生成** — `@extern(lib: "...", "...")` 宣言を生成
3. **C ABI 境界の型変換** — `--repr-c` でコンパイルされた shared library と Almide の型の間のマーシャリング（String ↔ C string、Record ↔ C struct）

型マッピングが identity なので、20 言語の中で最も実装が軽い。

## Scope

- **almide-bindgen 側**: 新しい `bindings/almide.almd` ジェネレーター（~200行見込み）
- **almide compiler 側**: `@extern(lib: "...")` の shared library 指定（既に `@extern` は存在、`lib:` パラメータ追加）
- **almide-lander 側**: `--lang almide` オプション追加（既存インフラ利用）
