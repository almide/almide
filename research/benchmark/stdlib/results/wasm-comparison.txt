Almide WASM (wasmtime):     6.66s user (222x vs native)

=== Breakdown ===

  WASM:   0.12s (6x vs native — wasmtime overhead, acceptable)
  Native: 0.02s

  WASM:   0.09s
  sort 100k extrapolated: ~6s (insertion sort O(n²))

Root cause: WASM list.sort uses insertion sort O(n²).
Fix: implement merge sort or quick sort in emit_wasm/calls_list_helpers.rs

=== WASM comparison v0.23.1 (post merge sort fix) ===

Almide WASM vs Rust WASM (wasmtime):
  fib(35) + sort 100k + map 100k: Almide ~0.15s vs Rust ~0.1s (1.5x — OK)
  
Remaining WASM bottlenecks:
  string concat: O(n²) — WASM has no string.push(mut) in-place mutation
  map insert: O(n²) — WASM has no map.insert(mut) in-place mutation
  
Root cause: WASM emitter uses immutable data structures.
mut operations (string.push, map.insert) need WASM-level COW or mutable heap.

=== WASM string/map analysis ===

Root cause: WASM string layout is [len:i32][data...] — fixed-size allocation.
Every concat allocates a new buffer. No in-place growth possible.

Options to fix:
1. Capacity-based layout: [len:i32][cap:i32][data...] — amortized O(1) append
   - Requires changing string layout across entire WASM emitter
   - All string operations need to respect len vs cap
   - list_layout.rs already has this pattern for lists

2. Peephole optimization: detect `s = s + expr` in while loop → accumulate in list + join
   - IR-level transformation, WASM target only
   - Doesn't require layout change
   - Limited to specific pattern

3. StringBuilder type: explicit mutable string buffer
   - User-visible API change
   - Already done for Rust target (string.push)

Recommendation: Option 1 (capacity-based layout) is the ideal long-term fix.

=== Loop 2 results ===
WASM map: still O(n) per set (full copy). Both map.set and map.insert
use the same copy path in WASM. Needs WASM-level in-place upsert with
capacity-based layout (already added header but no in-place logic yet).
This is a large refactor — map.set WASM implementation needs capacity-aware
rewrite (scan for existing key → overwrite or append with growth).
Tracked as future work.
