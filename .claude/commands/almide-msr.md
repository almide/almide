# MSR (Modification Survival Rate) Benchmark

Run the MSR benchmark: 25 Exercism-style exercises generated by a weak LLM (Haiku), compiled and tested by the current compiler.

## Steps

1. Run the MSR benchmark script against all 25 exercises:
   - For each `.almd.prompt` file in `research/benchmark/msr/prompts/`:
     - Use `claude --model haiku --print` with the CHEATSHEET as system prompt to generate code
     - Strip any markdown code fences from the output
     - Run `almide check` (type check)
     - Run `almide test` (test execution)
     - Track pass/fail counts
2. Report results: passed / total, check failures, test failures, MSR percentage
3. Save results JSON to `research/benchmark/msr/results/haiku_rust_{date}.json`
4. Save generated solutions to `research/benchmark/msr/outputs/haiku/`
5. Update README.md Project Status table with the new MSR number if it changed
6. Commit with message: `Update MSR benchmark: {passed}/{total} exercises pass with Haiku 4.5 on v{version}`

## System Prompt Template

Use `docs/CHEATSHEET.md` as language reference, plus these rules:
- Write ONLY the Almide code. No markdown, no explanation, no code fences.
- Replace `todo` with working implementations.
- Keep all test blocks exactly as they are.
- Keep all type declarations exactly as they are.
- Lambda syntax: `(x) => expr`, NOT `fn(x) => expr`
- `if` requires `then`: `if cond then a else b`
- String concat uses `+`
- Use `match` for pattern matching, not if chains on variants
- `effect fn` returns Result, auto-unwraps with `?`

## Notes

- Haiku is intentionally used as a weak model to measure language learnability
- Single-shot, no retry — measures first-attempt success rate
- wasm-smoke exercise tests WASM target (`almide test --target wasm`)
